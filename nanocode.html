AFD Autômato Finito Determinístico
Usado na análise léxica para reconhecer tokens. Cada caractere do código é processado por transições de estados. Ex: estado INICIO → lê letra → estado ID → lê dígito → estado ID (aceita identificador).
ER Expressões Regulares
Padrões que descrevem os tokens da linguagem. Identificadores: [a-z][a-z0-9]*. Números: [0-9]+. Equivalentes a AFDs pela hierarquia de Chomsky (Tipo 3).
CFL Gramática Livre de Contexto
Define a estrutura sintática da linguagem. Regras como stmt → ID = expr | print(expr). Reconhecida por Autômatos de Pilha (PDA) – Tipo 2 na hierarquia.
PDA Autômato de Pilha
Extensão do AFD com uma pilha. Permite verificar estruturas aninhadas e balanceamento. Usado no parser para validar a gramática livre de contexto.
CSL Linguagem Sensível ao Contexto
A análise semântica verifica propriedades que dependem do contexto, como variáveis definidas antes de serem usadas. Tipo 1 na hierarquia de Chomsky.
MT Máquina de Turing
Modelo teórico que executa sequencialmente as instruções. A execução do compilador simula uma MT: lê código intermediário, mantém estado (memória), e produz resultado. Tipo 0 – poder computacional máximo.
Hierarquia de Chomsky
Tipo 3 ← Expressões Regulares (Léxico) Tipo 2 ← Gramáticas Livres de Contexto (Sintático) Tipo 1 ← Gramáticas Sensíveis ao Contexto (Semântico) Tipo 0 ← Gramáticas Irrestritas (Execução / MT)
